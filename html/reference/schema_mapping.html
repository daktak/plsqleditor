Schema Mapping

Schema Mapping takes place in the Project Properties, as seen in this guide. There, you specify a:
  1. schema name
  2. directory/directories (and relative context) that contain files in that schema
  3. the database password for that schema. The connection details are assumed to be the default project setting ones found in the preferences, or overridden in the database connectivity properties of the project.
On a per file basis you can set the schema for a particular file by the following mechanisms:
  • Right clicking in the editor of the open file and selecting "Change Schema for Package"
  • Switching to the details tab in the editor and pressing the "Change Schema for Package" button on that tab.
  • In the top comment section (where the header details may be added as described in the howto guide you can add an @schema xxx tag, where xxx is the schema name
  • Manipulating the schema mapping file, which is located at /.metadata/.plugins/plsqleditor/{project_name}_schemaMappingsFile.store
    • .
However, there are easier techniques than manipulating the mapping file by file.

Auto Schema Mapping and Loading

In order to save having to set up an association per file, you can do the following:
  • Follow the Default naming convention. The default naming convention is that each package file will have the format SCHEMA_NAME DELIMITER PACKAGE_NAME DOT EXTENSION. The DELIMITER defaults to underscore. Doing this will mean every file you open will automatically be put in the right package, once a single file is opened in that directory, the rest will be auto-loaded.
  • how to override - The DELIMITER defaults to underscore (_) but can be modified in the preferences page (as schema to package delimiter) or per project in the project properties database connectivity page.
