This section covers the types of auto completion that you can manage in the Editor. There are two main types of auto completion -

  1. the coded auto completion
  2. the template driven auto completion (standard eclipse based templates)

Coded Auto Completion in the Editor

Auto completion has been coded to auto complete the following types of structures:
  • built in functions of plsql such as upper, lower, substr etc
  • schemas that are identified by the schema mapping
  • schemas that are detected by the connection to the database
  • packages that are detected by the schema mapping
  • packages that are detected by the connection to the database
  • procedures and functions that are detected in files identified in the schema mapping
  • procedures and functions that are detected by the connection to the database
  • tables and columns that are detected by the connection to the database
  • constants, local variables, procedure names (for end proc_name) etc that are in the local file being edited
These structures will be updated with the information in the files in your file system and in your database (if you connect to a database). Items not currently covered are:
  • types
  • array count, first, last, next
  • records and contained record fields
  • objects and their contained fields

Template based Auto Completion in the Editor

Template auto completion has been enabled in Toby's PL/SQL editor to allow users to determine their own auto completable structures. By default a new package body auto complete is present, along with new procedure, new function, begin end, for loops, while loops etc. You can modify the provided ones, and add new ones. There are two types of templates - code templates and pldoc templates.

Code templates
  • if constructs - if, if else, if elsif else
  • for loop constructs - for n, for array, for record in cursor
  • while loop constructs - basic loop while, standard loop over hash
  • new package body
  • new procedure, new function
  • begin exception block
  • savepoint

Document templates
  • true

You can add templates using the standard eclipse interface for managing templates. This site shows you how.
